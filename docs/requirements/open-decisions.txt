TECHNICAL DECISIONS
--------------------------------------------------

[OPEN] D1: Which voice model for MVP?
- Perplexity voice: best observed mispronunciation detection, API access unclear.
- GPT-4o Advanced Voice / Realtime: team has access, but weak pronunciation detection.
- Action: investigate Perplexity voice API access, pricing, and availability.
  If accessible, prototype with Perplexity. Otherwise, use GPT-4o Realtime.

[OPEN] D2: How to implement push-to-talk in TUI?
- Likely: hold a key (e.g., spacebar) to record audio, release to send.
- Needs a small spike to confirm this is feasible in a terminal environment.
- Action: prototype PTT audio capture in terminal this week.

[OPEN] D3: HSK vocabulary restriction — prompt-only or RAG-assisted?
- Option A: system prompt injection with HSK word list.
  - Risk: model may ignore or bypass constraint, especially for long lists.
  - HSK3 is ~600 words — fits in context window but needs testing.
- Option B: RAG — provide example sentences from allowed vocabulary to guide output.
- Option C: hybrid — prompt + post-generation vocabulary check.
- Action: test prompt-only first with HSK3 list; measure compliance rate.

[OPEN] D4: Backend language final choice?
- Team agreed: strongly-typed language required. No JavaScript. TypeScript rejected.
- Current prototype: Python (with type annotations via Pydantic, etc.)
- Alternatives: Go, Rust, or others.
- Action: decide whether to continue Python or switch before starting backend build.

[DECIDED] D5: Voice-to-voice vs STT+TTS for architecture preference?
- DECISION: Voice-to-voice preferred for pronunciation preservation.
- Fallback to STT+TTS only if voice model access is blocked.
- Details: see voice-architecture-requirements.txt

[DECIDED] D6: MVP platform?
- DECISION: TUI (terminal user interface) for MVP.
- Mobile app (React Native) is post-MVP, designed in Figma first.

[DECIDED] D7: Target language for MVP?
- DECISION: Mandarin Chinese, Simplified script only.
- "Chinese" is ambiguous — use "Mandarin" in all communications going forward.

[DECIDED] D8: Frontend framework for post-MVP mobile?
- DECISION: React Native (cross-platform iOS/Android).
- UI design in Figma before build.

[OPEN] D9: IPA pronunciation tokenization — pursue or park?
- Experimental idea: transcribe user speech to IPA symbols to preserve
  pronunciation even in non-voice-to-voice pipeline.
- Feasibility uncertain: LLMs may not interpret IPA well, tokenization issues.
- Action: park for now. Revisit only if voice-to-voice model access fails.

[OPEN] D10: Which Chinese AI model to consider for Mandarin-specific tasks?
- Hypothesis: a model trained primarily on Chinese data may outperform
  Western-centric LLMs for Mandarin pronunciation assessment.
- Referenced in call but no specific model named.
- Action: research Chinese-origin multimodal voice models (e.g., from Baidu, Alibaba,
  or others) and evaluate for pronunciation detection capability.
